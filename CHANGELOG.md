## Update Guidelines

When updating this project:

1. **Version Numbering**
   - MAJOR: Breaking changes, complete redesigns
   - MINOR: New features, sections, or capabilities
   - PATCH: Bug fixes, small improvements, content updates

2. **Changelog Entries**
   - Always date entries
   - Use clear, descriptive language
   - Group by type: Added, Changed, Deprecated, Removed, Fixed, Security
   - Link to issues/PRs when applicable

3. **Testing**
   - Test on multiple browsers
   - Verify mobile responsiveness
   - Check Lighthouse scores
   - Validate HTML/CSS
   - Test all links and interactions
